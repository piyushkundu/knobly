9.1 Introduction to User-Defined Functions

Functions are reusable blocks of code designed to perform a specific task. Python comes with many built-in functions like print(), len(), and max(), but you can also create your own functions, known as user-defined functions.

Why Use Functions?

  • Reusability: Write code once and use it multiple times
  • Modularity: Break complex problems into smaller, manageable pieces
  • Organization: Structure your code logically
  • Abstraction: Hide complex implementations behind simple interfaces
  • Maintainability: Update one function instead of multiple code blocks

Think of functions as "mini-programs" within your program that can be called whenever needed.

9.2 Defining a Function

In Python, functions are defined using the def keyword, followed by the function name and parentheses ().

def function_name(parameters):
    """Documentation string (docstring)"""
    # Function body - code that runs when function is called
    statement(s)
    return value  # Optional return statement

Example: Simple greeting function

def greet():
    """A simple function that prints a greeting message"""
    print("Hello, welcome to Python functions!")

Key components of a function definition:

  • Function header: Includes the def keyword, function name, parameters (if any), and colon.
  • Docstring: Optional but recommended documentation string that describes what the function does.
  • Function body: Contains the indented block of code that executes when the function is called.
  • Return statement: Optional statement that specifies the value to return from the function.

Best Practices for Function Names

  • Use lowercase letters with underscores (snake_case): calculate_average
  • Choose descriptive names that indicate what the function does
  • Avoid using reserved keywords as function names

9.3 Invoking a Function

To use a function, you need to call or "invoke" it by using its name followed by parentheses.

Example: Calling a function

def greet():
    print("Hello, welcome to Python functions!")

# Invoking the function
greet()
Output:
Hello, welcome to Python functions!

Functions can be called multiple times from different parts of your program:

def display_separator():
    print("-" * 30)

# Call the function three times
display_separator()
print("Section 1")
display_separator()
print("Section 2")
display_separator()

9.4 Function Arguments

Functions can accept inputs called arguments (or parameters) that allow them to work with different data each time they're called.

9.4.1 Positional Arguments

Positional arguments must be provided in the same order as they are defined in the function.

def greet_person(name, message):
    print(f"Hello, {name}! {message}")

# Calling with positional arguments
greet_person("Alice", "Welcome to Python.")
greet_person("Bob", "How are you today?")
Output:
Hello, Alice! Welcome to Python.
Hello, Bob! How are you today?

The first argument is assigned to the first parameter, the second argument to the second parameter, and so on.

9.4.2 Default Arguments

Default arguments have predefined values that are used when the corresponding argument is not provided in the function call.

def greet_person(name, message="Welcome to Python!"):
    print(f"Hello, {name}! {message}")

# Using default value for message
greet_person("Alice")
# Overriding default value
greet_person("Bob", "Glad to see you!")
Output:
Hello, Alice! Welcome to Python!
Hello, Bob! Glad to see you!

Important Rules for Default Arguments:

  • Default parameters must come after non-default parameters
  • Mutable objects (like lists, dictionaries) should not be used as default values (they are created only once when the function is defined)

9.4.3 Named Arguments

Named arguments (also called keyword arguments) allow you to specify parameter values by name, regardless of their position.

def display_info(name, age, city):
    print(f"{name} is {age} years old and lives in {city}.")

# Using named arguments
display_info(name="Alice", age=30, city="New York")
display_info(city="London", name="Bob", age=25)
display_info("Charlie", city="Paris", age=35)
Output:
Alice is 30 years old and lives in New York.
Bob is 25 years old and lives in London.
Charlie is 35 years old and lives in Paris.

Benefits of named arguments:

  • Makes function calls more readable and self-documenting
  • Allows you to specify arguments in any order
  • Reduces errors from mixing up argument order

Note: You can mix positional and named arguments, but positional arguments must come before any named arguments.

9.5 Types of Functions

Functions can be categorized based on whether they accept inputs (parameters) and return outputs.

9.5.1 Function with Input and Output

These functions accept parameters and return values. They're ideal for transforming data or performing calculations.

def calculate_area(length, width):
    """Calculate the area of a rectangle"""
    area = length * width
    return area

# Call the function and store the returned value
rectangle_area = calculate_area(5, 3)
print(f"The area of the rectangle is {rectangle_area} square units")

# Use the returned value directly in another calculation
perimeter = 2 * (5 + 3)
print(f"Area to perimeter ratio: {rectangle_area / perimeter}")
Output:
The area of the rectangle is 15 square units
Area to perimeter ratio: 0.9375

These functions are highly reusable and can be part of larger expressions.

9.5.2 Function with Input and without Output

These functions accept parameters but don't explicitly return a value (or implicitly return None). They're typically used for actions or side effects.

def print_details(name, age):
    """Display formatted user details"""
    print(f"Name: {name}")
    print(f"Age: {age}")
    print(f"Birth year: {2024 - age} (approximately)")
    # No return statement

# Call the function
print_details("Alice", 30)

# If you try to capture a return value, you'll get None
result = print_details("Bob", 25)
print(f"Return value: {result}")
Output:
Name: Alice
Age: 30
Birth year: 1994 (approximately)
Name: Bob
Age: 25
Birth year: 1999 (approximately)
Return value: None

These functions are useful for tasks like displaying information, writing to files, or updating databases.

9.5.3 Function without Input and with Output

These functions don't take parameters but return a value. They often generate values or retrieve information from elsewhere.

import random
import time

def get_random_number():
    """Generate a random number between 1 and 100"""
    return random.randint(1, 100)

def get_current_time():
    """Return the current time as a string"""
    return time.strftime("%H:%M:%S")

# Call the functions
random_number = get_random_number()
print(f"Random number: {random_number}")

current_time = get_current_time()
print(f"Current time: {current_time}")
Output:
Random number: 42
Current time: 14:35:27

These functions are useful for generating data, retrieving system information, or providing utility values.

9.5.4 Function without Input or Output

These functions neither accept parameters nor return values. They perform a specific action each time they're called.

def display_menu():
    """Display the application menu"""
    print("\n===== MAIN MENU =====")
    print("1. Start new game")
    print("2. Load saved game")
    print("3. Settings")
    print("4. Exit")
    print("=====================\n")

def display_separator():
    """Print a separator line"""
    print("-" * 30)

# Call the functions
display_separator()
display_menu()
display_separator()
Output:
------------------------------

1. Start new game
2. Load saved game
3. Settings
4. Exit
=====================

------------------------------

These functions are useful for organizing code that performs standard operations with no variable inputs.

9.6 Scope of Variables

Variable scope refers to the region of a program where a variable is accessible. Python has four types of variable scopes:

  • Local scope: Variables defined within a function
  • Enclosing scope: Variables in the outer function (in nested functions)
  • Global scope: Variables defined at the top level of a module
  • Built-in scope: Names that are pre-defined in Python

Example: Local vs. Global variables

global_var = "I'm a global variable"

def function_with_local_var():
    local_var = "I'm a local variable"
    print(local_var)      # Accessing local variable
    print(global_var)     # Accessing global variable

function_with_local_var()
print(global_var)         # Accessing global variable
# print(local_var)        # This would cause an error - local_var is not accessible here
Output:
I'm a local variable
I'm a global variable
I'm a global variable

The global Keyword

To modify a global variable from within a function, you need to use the global keyword:

counter = 0

def increment_counter():
    global counter    # Declare we want to use the global variable
    counter += 1
    print(f"Counter inside function: {counter}")

print(f"Counter before: {counter}")
increment_counter()
print(f"Counter after: {counter}")
Output:
Counter before: 0
Counter inside function: 1
Counter after: 1

The nonlocal Keyword

For nested functions, the nonlocal keyword allows you to modify a variable from the enclosing scope:

def outer_function():
    outer_var = "I'm in the outer function"
    
    def inner_function():
        nonlocal outer_var
        outer_var = "Modified by inner function"
        print(f"Inside inner function: {outer_var}")
    
    print(f"Before inner call: {outer_var}")
    inner_function()
    print(f"After inner call: {outer_var}")

outer_function()
Output:
Before inner call: I'm in the outer function
Inside inner function: Modified by inner function
After inner call: Modified by inner function

Variable Lookup Order (LEGB Rule)

Python searches for variables in this order:

  1. Local scope
  2. Enclosing scope
  3. Global scope
  4. Built-in scope

9.7 Recursion

Recursion is a programming technique where a function calls itself to solve a problem. It's particularly useful for problems that can be divided into similar sub-problems.

Example: Factorial calculation using recursion

def factorial(n):
    """Calculate the factorial of a number using recursion"""
    # Base case
    if n == 0 or n == 1:
        return 1
    # Recursive case
    else:
        return n * factorial(n-1)

# Test the function
print(f"Factorial of 5: {factorial(5)}")
print(f"Factorial of 0: {factorial(0)}")
Output:
Factorial of 5: 120
Factorial of 0: 1

Every recursive function has two key components:

  • Base case: The condition that stops the recursion
  • Recursive case: The function calling itself with a smaller instance of the problem

Example: Fibonacci sequence using recursion

def fibonacci(n):
    """Return the nth number in the Fibonacci sequence"""
    # Base cases
    if n <= 0:
        return 0
    elif n == 1:
        return 1
    # Recursive case
    else:
        return fibonacci(n-1) + fibonacci(n-2)

# Generate first 10 Fibonacci numbers
for i in range(10):
    print(f"Fibonacci({i}) = {fibonacci(i)}")
Output:
Fibonacci(0) = 0
Fibonacci(1) = 1
Fibonacci(2) = 1
Fibonacci(3) = 2
Fibonacci(4) = 3
Fibonacci(5) = 5
Fibonacci(6) = 8
Fibonacci(7) = 13
Fibonacci(8) = 21
Fibonacci(9) = 34

Recursion Considerations

  • Memory usage: Each recursive call adds a new frame to the call stack
  • Stack overflow: Too many recursive calls can exceed the maximum stack depth
  • Performance: Recursive solutions can be less efficient than iterative approaches (e.g., naive recursive Fibonacci)
  • Optimization: Techniques like memoization can improve recursive performance

Example: Optimized Fibonacci with memoization

def fibonacci_memoized(n, memo=None):
    """Optimized Fibonacci using memoization to avoid redundant calculations"""
    if memo is None:
        memo = {}
    
    # Check if we've already calculated this value
    if n in memo:
        return memo[n]
    
    # Base cases
    if n <= 0:
        return 0
    elif n == 1:
        return 1
    
    # Calculate and store the result
    memo[n] = fibonacci_memoized(n-1, memo) + fibonacci_memoized(n-2, memo)
    return memo[n]

# Much faster for larger values
print(f"Fibonacci(30) = {fibonacci_memoized(30)}")
Output:
Fibonacci(30) = 832040
